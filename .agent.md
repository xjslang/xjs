## Goals

**The current priority is to keep the code as clear and simple as possible.** For this reason, the system will not act in a "paternalistic" way and will allow users to write syntactically valid expressions and statements even if they are semantically incorrect. Otherwise, we would have to consider all edge cases and add complexity to the code.

Additionally, utilities like ESLint already exist to serve that purpose.

For example:

```go
func Test(t *testing.T) {
	// semantically incorrect
	input := `a + b = 100`

	// the code above does not generate errors, because it is understood
	// that writing semantically correct code is the user's responsibility.
	result, err := xjs.Parse([]byte(input))
	require.NoError(t, err)
}
```
